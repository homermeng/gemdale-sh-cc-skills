# 金地上海地区公司市场营销部，案场客户接访数据分析和客户分析日报输出

## 任务背景
金地上海地区公司在售各房地产项目，使用智慧工牌（所谓智慧工牌，即销售人员佩戴在胸前的装有录音设备的工牌装置，该装置能够在销售人员接待来访购房客户的介绍、讲解过程中，全程对销售人员与客户的对话进行记录，后台ASR模型将语音记录转写为文本，形成接访事件的文字模态数据记录）记录项目案场销售人员接待来访客户的讲解和对话内容，形成接访数据。

## 输入数据

用户将提供某一项目，某日的若干次接访数据（已由ASR转写为文本模态），这些数据存放于飞书云盘的指定文件夹，用户将提供目标飞书文件夹的token，调用`./scripts/feishu_download_files.py`脚本，将用户指定文件夹下的所有文件下载至项目目录的子文件夹下（具体子文件夹名称用户也会提供，若用户未提供，则询问用户）。下载得到的数据，格式可能是docx、xlsx、pdf、txt等格式的文件，结合具体情况，调用相关技能或撰写临时代码读取下载的各数据文件。

## 客户来访分析步骤和要求

### 阶段二：汇总当日来访客户的总体情况，撰写并生成案场客户来访分析日报
在阶段一生成的“yyyy-mm-dd_项目名称_客户来访信息提取.md”数据文件基础上，对当日来访的n组客户情况进行总体分析，并根据内容和业务要求生成和输出案场客户来访分析日报。

对于报告的各个部分（各章节标题和内容逻辑固定），内容和业务要求如下：

```
# 当日结论摘要

列表展示当日来访客户的：1、客户分级；2、核心关注点；3、竞品对比；4、产品满意度；5、价格承受度。
表格为两列6行，第一行为列名“维度”、“核心结论”，2~6行为上述5个维度的当日来访客户情况总体总结和输出。
示例：
|维度	|核心结论
|1. 客户分级	|高意向占75%（6/8组），中意向25%（2/8组），无低意向；改善型为主，100%聚焦大户型
|2. 核心关注点	|价格承受力＞产品细节＞政策解读；地下室利用、楼间距、落地窗设计为三大产品痛点
|3. 竞品对比	|万科云间传奇为主竞；我方赠送面积近50%为绝对优势，九龙仓/中铁建因"价高户型差"被动出局
|4. 产品满意度	|不满：楼间距近（4次）、落地窗设计（安全/节能/私密性）；满意：院子实用性、赠送面积
|5. 价格承受度	|明确预算仅1组（900万），首付缺口100万；87.5%客户未披露总价上限，深度排摸不足

注意：在总结当日来访客户的5维度特征时，务必提取核心结论，不必铺陈细节，这个章节主要面向公司管理者，需要在一张表上让读者快速抓住当日来访客户的总体状态，以快速把握和感受市场、客户对该项目的总体反应。

# 一、当天接待客户分级

文字描述当日来访客户的意向度分级情况，客户意向度分为高、中、低三级，高意向度客户在与销售员互动的过程中，表现出更主动询问问题，关心价格和交付时间；客户有购房资格，购房原因合理，资金准备充足等；而中低意向度客户则是在上述维度中某个或某几个条件不具备。

示例：
高意向客户（6组，75%）：特征为家住附近、置换需求迫切、预算相对明确，已进入深度比较阶段。
中意向客户（2组，25%）：特征为仍在广泛看盘对比，对产品有个别明显顾虑（如楼间距）。
低意向客户：0组。
- 行动建议：
    - 集中资源优先跟进6组高意向客户，解决其关键决策障碍（如资金周转、产品疑虑），加速转化。

# 二、客户核心关注点

文字描述从当日来访客户数据中所反映出的，客户对于该项目的核心关注点，该关注点可能是价格，可能是产品（具体的产品维度，如两面宽的上叠产品次卫是暗卫等具体的关注点、抗性点等）、可能是周边配套等、学区配置等等。

示例：
价值点关注排序：
排序1价格与价值：总价、首付、得房率、赠送面积。
排序2产品硬指标：房间尺寸、装修标准、楼间距。
排序3生活方式附加值：地下室利用、院子大小、露台、采光设计。
- 行动建议：
    - 营销说辞与物料强化“高赠送”和“精装标准”，直观展示性价比。

# 三、竞品提及与竞争力评估

列表展示当日来访客户在与销售员对话过程中，提到的竞品项目，以及客户对竞品项目的评价，竞品项目与该项目对比下的优劣势等。
表格为4列，多行，四列分别为：竞品项目、提及率、客户评价、我方相对优势；提及率的计算，即当日来访总客户n组中，有m组提及了某竞品项目，则该竞品项目的“提及率”几位m/n（输出为百分数）。

示例：
|竞品项目       	 |提及率   	 |客户评价         	 |我方相对优势  
|万科云间传奇	|37.50%	|在考虑范围 	 |赠送面积碾压  
|九龙仓	|12.50%	|价高、房型差	 |价格+户型双优 
|丰盛道 	|12.50%	|我方更好  	 |已形成认知优势 
|中铁建	|12.50%	|价格偏高	 |性价比突出   
|其他二手房	|25%	|便宜但房龄老	|新房品质溢价  

列表展示后，提供简要的行动建议，示例：

- 行动建议：
    - 1.	制作《核心竞品对比工具卡》，重点突显我方在得房率、赠送面积上的压倒性优势。
    - 2.	针对万科项目，一线销售需熟练掌握“产品力（户型实用率）VS 品牌力”的对比话术。

# 四、产品认可度与主要不满意点

文字和表格总结当日来访客户对该项目满意的地方，以及不满意的地方（重点）；满意点文字简要总结即可，不满意的地方，列表展示；
表格为3列多行，列名为：问题类别、提及次数、不满点；
满意点和不满意点描述后，提供简要的行动建议。

本章节整体示例：

**满意点：**
赠送面积、项目整体品质、优于二手房的新房体验
**不满意点：**
|问题类别	|提及次数	|不满点
|楼间距	|4 	|过近
|落地窗设计	|1 	|担心自爆、节能、私密性
|电梯/户型布局	|1 	|电梯小
|院子	|1 	|偏小
下叠产品接受度复杂	50%的提及客户有兴趣但存在顾虑（面积紧凑、打理麻烦、地下室潮湿）
- 行动建议：
    - 统一解释口径：准备楼间距规划文件，强调合规性与对采光通风的保障。
    - 准备解决方案：针对落地窗，展示玻璃质量认证与私密性处理案例（如镀膜）。针对地下室，提供防潮方案和空间利用建议（如影音室、健身房）。

# 五、价格承受力排摸

文字描述和总结当日来访客户的资金准备情况、价格承受能力等情况，示例

总价区间：仅有1组客户明确预算控制在900万左右，并倾向精装修交付。
普遍心态：大部分客户对总价敏感，但更关注单位面积价格（性价比）。首付压力是主要瓶颈（1组客户首付差100万）。
价格敏感度：客户主动进行四类比价——竞品质价、二手房价差、政策优惠、付款方式灵活性。
- 行动建议：
    - 强化价值包装：在洽谈中多使用“折算后单价”来计算价值。
    - 灵活付款方案：对有意向但资金暂有困难的客户，联合银行定制"首付分期"方案，旧房置换协助等解决方案。

注意：“销售”指销售员的姓名或代号，从原始数据文件的文件名中可以获取得到，其他方面的表现，请从原始接访数据中进行分析和提炼、评价。

# 七、今日成交障碍

总结今日来访客户未能成交的主要原因、阻碍，如对产品不满意、对区位不满意、价格太高、家庭决策人不在场等等，示例：

- 1.	产品抗性：楼间距近（高频）、落地窗设计疑虑、下叠接受度复杂。

注意：避免大而空的泛泛而谈，提建议时，务必针对具体问题，提出能够快速改变和执行的建议；严格避免讲正确的废话，如“提高销售员培训”等类似的建议。
```
